# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Vite

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/main.ts — точка входа приложения

- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
В проекте используются строго типизированные данные, описанные через TypeScript-интерфейсы.

#### Интерфейс IProduct
Интерфейс описывает структуру данных товара.

Поля интерфейса:
`id: string` - уникальный идентификатор товара
`title: string` - название товара
`price: number | null` - цена товара. Принимает число или null, если цена не указана
`description: string` - описание товара
`image: string` - путь к изображению товара
`category: string` - категория товара

#### Интерфейс IBuyer
Описывает структуру данных покупателя. Используется при оформлении заказа.

Поля интерфейса:
`payment: TPayment` - способ оплаты
`email: string` - email покупателя
`phone: string` - телефон покупателя
`address: string` - адрес доставки

#### Тип TPayment
Тип описывает допустимые значения способа оплаты.
Принимает одно из значений: 'card', 'cash' или '' (пустая строка, если способ не выбран).

#### FormError
Частичный объект (не все поля обязательны), где ключи - это названия полей из интерфейса IBuyer (payment, email, phone, address), а значения - строки с сообщениями об ошибках.

### Модели данных
Модели инкапсулируют данные и предоставляют безопасный доступ через методы.
Они не зависят от UI, API или DOM.

#### Класс Catalog
Хранит список всех доступных товаров и информацию о выбранном товаре (для детального просмотра).

Поля класса:
`productList: IProduct[]` - массив всех товаров, доступных в каталоге
`selectedProduct: IProduct | null` - текущий выбранный товар. Может быть null, если товар не выбран

Методы класса:
`setProducts(products: IProduct[]): void` - сохраняет массив товаров, переданный в параметре, в поле `productList`
`getProducts(): IProduct[]` - возвращает копию массива всех товаров
`getProductsById(id: string): IProduct | null` - находит товар по id; возвращает товар или `null`, если не найден
`setSelectedProduct(product: IProduct | null): void` - устанавливает переданный товар как выбранный
`getSelectedProducts(): IProduct | null` - возвращает копию выбранного товара или `null`

#### Класс Bucket
Хранит товары, выбранные пользователем для покупки.

Поля класса:
`products: IProduct[]` - массив товаров в корзине
Методы класса:
`getProducts(): IProduct[]` - возвращает копию массива товаров в корзине
`addProduct(product: IProduct): void` - добавляет товар в корзину
`removeProduct(id: string): void` - удаляет товар из корзины по id
`clear(): void` - очищает корзину
`getTotalPrice(): number` - возвращает общую стоимость товаров (значения `null` трактуются как 0)
`getCount(): number` - возвращает количество товаров в корзине
`hasProduct(id: string): boolean` - проверяет наличие товара в корзине по `id`

#### Класс Buyer
Модель покупателя отвечает за хранение и валидацию контактных данных для оформления заказа.

Поля класса:
`data: IBuyer` - объект с данными покупателя:
  -`payment: TPayment` - способ оплаты ('card', 'cash' или '')
  -`address: string` - адрес доставки
  -`phone: string` - телефон
  -`email: string` - электронная почта

Методы класса:
`updateData(partialData: Partial<IBuyer>): void` - обновляет любые поля данных частично, не затирая остальные
`getData(): IBuyer` - возвращает копию всех данных покупателя
`clear(): void` - сбрасывает все поля в начальное состояние
`validate(): FormError` - проверяет, что все поля заполнены. Возвращает объект ошибок, где ключи - имена полей, значения - сообщения об ошибках. Поле считается валидным, если оно не пустое (после удаления пробелов).

### Слой коммуникации
Слой коммуникации отвечает за обмен данными между приложением и сервером через API. Класс этого слоя использует композицию для работы с базовым классом Api.

#### Интерфейс IProductList
Описывает структуру ответа сервера при запросе каталога товаров.
Поля интерфейса:
`total: number` - общее количество товаров на сервере
`items: IProduct[]` - массив товаров

#### Интерфейс IOrderRequest
Описывает структуру данных, отправляемых на сервер при оформлении заказа.

Поля интерфейса:
`payment: TPayment` - способ оплаты ('card' или 'cash')
`address: string` - адрес доставки
`email: string` - email покупателя
`phone: string` - телефон покупателя
`items: string[]` - массив id товаров из корзины

#### Интерфейс IOrderResponse
Интерфейс описывает структуру ответа, получаемого от сервера после успешного оформления заказа.

Поля интерфейса:
`id: string` — уникальный идентификатор созданного заказа (например, "ord-854cef69").
`total: number` — итоговая сумма заказа в рублях, рассчитанная на основе цен товаров из корзины.

#### Класс LarekAPI
Класс представляет слой коммуникации с сервером.

Конструктор:
`constructor(api: IApi)` - принимает объект, реализующий интерфейс IApi

Методы класса:
`getProducts(): Promise<IProduct[]>` - выполняет GET-запрос к эндпоинту /product/, получает ответ типа IProductList и возвращает массив items
`sendOrder(order: IOrderRequest): Promise<void>` - выполняет POST-запрос к эндпоинту /order/ с данными заказа

### Слой Представления (View) -----------------------------------------------------------
Слой представления отвечает за то, как данные показываются на экране и как пользователь может с ними взаимодействовать. Все элементы этого слоя происходят от базового класса `Component` и используют систему событий, чтобы сообщать о действиях пользователя.

#### Класс Card
Базовый класс для всех карточек товаров. Включает основные функции для работы с карточками товаров.

Наследование
`Component` → `Card` → `CardCatalog`, `CardPreview`, `CardBasket`

Конструктор:
Принимает два параметра: `container` типа `HTMLElement` для корневого DOM-элемента карточки и `events` типа `IEvents` для экземпляра брокера событий.

Поля класса:
Поле `_title` типа `HTMLElement` с модификатором _protected_. DOM-элемент для отображения названия товара с селектором `.card__title.`
Поле `_price` типа `HTMLElement` с модификатором _protected_. DOM-элемент для отображения цены товара с селектором `.card__price.`
Поле `events` типа `IEvents` с модификатором _protected_. Ссылка на брокер событий, передаётся из конструктора.

Методы класса
Метод `setTitle` принимает параметр `title` типа _string_ и устанавливает текст заголовка карточки.
Метод `setPrice` принимает параметр `price` типа _number_ или _null_ и устанавливает текст цены карточки. Если цена равна _null_, отображается текст Бесценно, иначе отображается значение цены с суффиксом синапсов.

События
Класс `Card` сам не генерирует события, но предоставляет инфраструктуру `events` для подклассов.

#### Класс CardImage
Абстрактный класс для карточек товаров с изображением. Расширяет базовый класс `Card` функционалом для работы с категориями и изображениями.

Наследование
`Component` → `Card` → `CardImage` → `CardCatalog`, `CardPreview`

Конструктор
Принимает два параметра: `container` типа _HTMLElement_ для корневого DOM-элемента карточки и `events` типа _IEvents_ для экземпляра брокера событий.

Поля класса
Поле `_category` типа _HTMLElement_ с модификатором protected. DOM-элемент для отображения категории товара с селектором `.card__category.`
Поле `_image` типа _HTMLImageElement_ с модификатором protected. DOM-элемент для отображения изображения товара с селектором `.card__image.`

Методы класса
Метод `setCategory` принимает параметр `category` типа _string_ и устанавливает текст категории, а также добавляет соответствующий класс модификатора из `categoryMap`.
Метод `setProductImage` принимает параметры _src_ типа _string_ и _alt_ типа _string_ (опционально). Устанавливает изображение с префиксом `CDN_URL` через метод `setImage` родительского класса.

События
Класс не генерирует события самостоятельно, предоставляет инфраструктуру для подклассов.

#### Класс CardCatalog
Компонент карточки товара для отображения в каталоге на главной странице. Поддерживает клик для открытия детального просмотра. Наследуется от `CardImage`.

Наследование
`Component` → `Card` → `CardWithImage` → `CardCatalog`

Конструктор:
Принимает два параметра: `container` типа _HTMLElement_ и `events` типа _IEvents_. Регистрирует обработчик клика на контейнер.

Поля класса:
- Все поля наследуются от `Card` и `CardWithImage`
- `_clickHandler?: () => void` - приватный колбэк для обработки клика

Методы класса:
- `render(data?: Partial<IProduct>): HTMLElement` - отображает карточку товара в каталоге с данными из параметра. Создает колбэк с данными товара через замыкание. При клике на карточку генерирует событие `'card:select'` с данными товара из колбэка

События
Генерирует событие `card:select` с данными { product: IProduct } при клике на карточку.

#### Класс CardPreview
Карточка товара для отображения в модальном окне просмотра. Содержит кнопку добавления в корзину или удаления из корзины.

Наследование
`Component` → `Card` → `CardWithImage` → `CardPreview`

Конструктор
Принимает два параметра: `container` типа _HTMLElement_ и `events` типа _IEvents_. Находит элементы описания и кнопки, регистрирует обработчик клика.
Поля класса
Поле `_text` типа _HTMLElement_ с модификатором protected. DOM-элемент для описания товара с селектором `.card__text.`
Поле `_button` типа HTMLButtonElement с модификатором protected. DOM-элемент кнопки действия с селектором `.card__button.`
Поле `_buttonClickHandler` типа () => void с модификатором _private_. Обработчик клика на кнопку.

Методы класса
Метод `render` принимает параметр `data` типа `Partial<IProduct & { inBasket: boolean }>`. Устанавливает все поля товара и описание. Управляет состоянием кнопки в зависимости от наличия товара в корзине и доступности цены.

События
Генерирует событие `card:add` с данными `{ product: IProduct }` при нажатии кнопки В корзину.
Генерирует событие `card:remove` с данными `{ product: IProduct }` при нажатии кнопки Удалить из корзины.

#### Класс CardBasket
Карточка товара для отображения в корзине покупок. Содержит индекс товара и кнопку удаления.

Наследование
`Component` → `Card` → `CardBasket`

Конструктор
Принимает два параметра: `container` типа _HTMLElement_ и `events` типа _IEvents_. Находит элементы индекса и кнопки удаления, регистрирует обработчик клика.

Поля класса
Поле `_index` типа _HTMLElement_ с модификатором _protected_. DOM-элемент для отображения номера товара в корзине с селектором .basket__item-index.
Поле `_button` типа _HTMLButtonElement_ с модификатором _protected_. DOM-элемент кнопки удаления с селектором .basket__item-delete.
Поле `_buttonClickHandler` типа () => void с модификатором _private_. Обработчик клика на кнопку удаления.
Методы класса
Метод render принимает параметр data типа `Partial<IProduct & { index: number }>`. Устанавливает индекс, заголовок и цену товара. Создаёт колбэк для события удаления.

События
Генерирует событие `card:remove` с данными `{ product: IProduct }` при нажатии кнопки удаления.

#### Базовый класс Form
Абстрактный базовый класс для всех форм приложения. Предоставляет общую функциональность для работы с формами.

Наследование
`Component` → `Form` → `OrderForm`, `ContactsForm`

Конструктор
Принимает два параметра: `container` типа _HTMLElement_ и `events` типа _IEvents_. Находит кнопку отправки и элемент ошибок.

Поля класса
Поле `_submitButton` типа HTMLButtonElement с модификатором _protected_. DOM-элемент кнопки отправки с селектором `button[type="submit"].`
Поле `_errors` типа _HTMLElement_ с модификатором _protected_. DOM-элемент для отображения ошибок с селектором `.form__errors.`
Поле `events` типа _IEvents_ с модификатором _protected_. Ссылка на брокер событий.

Методы класса
Метод `setErrors` принимает параметр `errors` типа _string[]_. Отображает ошибки через запятую или очищает поле.
Метод `setSubmitButtonDisabled` принимает параметр `disabled` типа _boolean_. Управляет состоянием кнопки отправки.
Метод `reset` очищает ошибки и блокирует кнопку отправки.

События
Класс не генерирует события самостоятельно, предоставляет инфраструктуру для подклассов.

#### Класс OrderForm
Форма для первого шага оформления заказа. Выбор способа оплаты и ввод адреса доставки.

Наследование
`Component` → `Form` → `OrderForm`

Конструктор
Принимает два параметра: `container` типа HTMLElement и `events` типа IEvents. Находит кнопки оплаты и поле адреса, регистрирует обработчики событий.

Поля класса
Поле `_paymentButtons` типа _HTMLButtonElement[]_ с модификатором _protected_. Массив кнопок выбора способа оплаты с селектором .order__buttons button.

#### Класс ContactsForm
Форма для ввода контактных данных покупателя на втором шаге оформления заказа. Включает валидацию email и телефона, фильтрацию ввода в реальном времени и очистку полей.

**Наследование**
`Component` → `Form` → `ContactsForm`

Конструктор
Принимает два параметра: `container` типа `HTMLElement` для корневого DOM-элемента формы и `events` типа `IEvents` для экземпляра брокера событий. Находит поля email и телефона, настраивает их типы и паттерны валидации, регистрирует обработчики ввода и отправки формы.

Поля класса
Поле `_emailInput` типа `HTMLInputElement` с модификатором _protected_. DOM-элемент поля ввода email с селектором `input[name="email"]`. Настроено с типом `email`, паттерном для валидации формата и обязательным заполнением.

Поле `_phoneInput` типа `HTMLInputElement` с модификатором _protected_. DOM-элемент поля ввода телефона с селектором `input[name="phone"]`. Настроено с типом `tel`, паттерном для допуска только цифр и символов форматирования, обязательным заполнением.

Методы класса
Метод `reset` очищает значения полей email и телефона, вызывает базовый метод очистки формы.

Метод `render` принимает параметр `data` типа `Partial<IBuyer>`. Заполняет поля формы значениями из переданных данных, если они присутствуют. Возвращает корневой элемент формы.

Обработчики событий ввода
Обработчик input на поле email проверяет валидность значения через `checkValidity()` и генерирует событие `contacts:change` только при успешной валидации.

Обработчик input на поле телефона фильтрует ввод, оставляя только цифры и разрешённые символы форматирования, затем проверяет валидность и генерирует событие `contacts:change` при успешной валидации.

Обработчик submit на контейнере формы предотвращает стандартную отправку, проверяет валидность обоих полей и генерирует событие `contacts:submit` только при успешной валидации, иначе вызывает `reportValidity()` для показа сообщений браузера.

События
Генерирует событие `contacts:change` с данными `Partial<IBuyer>` при изменении и успешной валидации полей email или телефона.

Генерирует событие `contacts:submit` при отправке формы после успешной валидации всех полей.

#### Класс Modal
Компонент модального окна для отображения динамического контента. Управляет открытием, закрытием и заменой содержимого, блокирует скролл страницы при активном состоянии.

Наследование
`Component` → `Modal`

Конструктор
Принимает параметр `container` типа `HTMLElement` для корневого DOM-элемента модального окна. Находит элементы контента и кнопки закрытия, регистрирует обработчики клика для закрытия по крестику или клику вне области, а также обработчик скролла для предотвращения прокрутки модального окна.

Поля класса
Поле `_content` типа `HTMLElement` с модификатором _protected_. DOM-элемент контейнера для динамического контента с селектором `.modal__content`.
Поле `_closeButton` типа `HTMLButtonElement` с модификатором _protected_. DOM-элемент кнопки закрытия с селектором `.modal__close`.

Методы класса
Метод `setContent` принимает параметр `content` типа `HTMLElement`. Заменяет текущее содержимое модального окна на переданный элемент с помощью `replaceChildren`.
Метод `open` добавляет класс `modal_active` к корневому элементу для отображения модального окна и устанавливает `overflow: hidden` для `document.body` для блокировки прокрутки страницы.
Метод `close` убирает класс `modal_active`, восстанавливает прокрутку страницы очисткой стиля `overflow` и очищает содержимое модального окна через `innerHTML`.
Метод `containsBasket` возвращает значение типа _boolean_. Проверяет наличие элемента с классом `.basket` внутри контента модального окна.
Метод `render` возвращает корневой DOM-элемент контейнера.

Обработчики событий
Обработчик `click` на кнопке закрытия вызывает метод `close` для закрытия модального окна.
Обработчик `click` на корневом контейнере проверяет, был ли клик произведён непосредственно по контейнеру (а не по его дочерним элементам), и вызывает `close` для закрытия при клике вне области контента.
Обработчик `wheel` на контейнере вызывает `stopPropagation` для предотвращения прокрутки содержимого модального окна при наведении курсора.

События
Класс `Modal` не генерирует события самостоятельно. Закрытие модального окна инициируется пользовательскими действиями (клик на крестик или вне области), которые обрабатываются внутри класса. Внешние компоненты могут отслеживать состояние модального окна через проверку контента или подписку на события дочерних компонентов, размещённых внутри.

### Презентер

Презентер в приложении отвечает за основную логику и обеспечивает взаимодействие между слоями данных и отображения. В этом проекте презентер реализован в файле `main.ts` через обработчики событий.

Он следит за событиями, поступающими от моделей данных и элементов интерфейса, обрабатывает их и запускает соответствующие методы моделей или компонентов для обновления состояния приложения.

Основные функции презентера включают:

- Инициализацию приложения: создание экземпляров моделей, компонентов интерфейса и API.

- Обработку событий от моделей данных: обновление интерфейса при изменении данных.

- Обработку событий от компонентов интерфейса: выполнение действий с данными и обновление интерфейса.

- Управление диалоговыми окнами: открытие и закрытие диалоговых окон в зависимости от действий пользователя.

- Проверку данных: валидацию информации в формах перед отправкой заказа.

- Отправку заказа на сервер: подготовку данных заказа и их передачу через API.

Презентер не инициирует события, а лишь обрабатывает их, обеспечивая взаимосвязь между слоями приложения в соответствии с паттерном MVP.
